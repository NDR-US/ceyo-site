# Support

## Documentation

The README provides an overview of the repository and its contents.

Protocol specifications, architecture documents, and formal schema definitions are maintained in the separate `ceyo-protocol` repository.

---

## Reporting Issues

If you encounter a bug or documentation issue, please open a GitHub Issue and include:

• A clear description of the problem
• Steps to reproduce the issue
• Expected vs actual behavior

---

## Security Issues

Security vulnerabilities should be reported through the process described in `SECURITY.md`.

Please avoid publicly disclosing security issues before they are reviewed.

---

## Feature Requests

Suggestions and improvements are welcome.

Feature proposals should explain:

• The problem being addressed
• Why the change improves the protocol or implementation
• Potential compatibility implications
